# Contribution Guide

## Before contributing

- Use Node.js 18+ and npm 9+.
- Work from the repository root.
- Avoid committing generated files from `backend/uploads/` or `backend/outputs/`.

## Recommended workflow

1. Create a descriptive branch.
2. Keep each change focused on one purpose.
3. Update both backend and frontend when a change affects the shared flow.
4. Verify the UI and conversion behavior manually.

## Pull request checklist

- Describe the problem being solved.
- Explain which part of the flow changed.
- Mention how you tested the change.
- Include screenshots if the UI changed.
