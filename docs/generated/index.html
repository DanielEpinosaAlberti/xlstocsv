API contracts

POST /api/analyze

Accepts multipart form data with a file field and returns workbook metadata.

{
  "success": true,
  "headers": ["Name", "Date", "Amount"],
  "sampleRows": [{ "Name": "José", "Date": "25/03/2026", "Amount": "1.234,56" }],
  "totalRows": 150
}

POST /api/convert

Accepts a spreadsheet file plus JSON-encoded normalization options and column rules.

{
  "success": true,
  "downloadUrl": "/outputs/filename_1711180800000.csv",
  "preview": {
    "headers": ["name", "date", "amount"],
    "rows": [{ "name": "jose", "date": "2026-03-25", "amount": "1234.56" }],
    "totalRows": 150
  }
}

Normalization

The documented utility functions transform headers and values before CSV export.

  • normalizeHeader(header) converts headers to snake_case and removes accents.
  • normalizeValue(value, options) applies global text, date, accent, and number normalization.
  • applyColumnRules(value, rules) applies per-column overrides.
  • removeAccents(str) strips accent marks from a string.
Per-column rules take precedence over the global normalization flags for the columns that are explicitly configured.

Data flow

  1. User uploads a spreadsheet in the frontend.
  2. The backend stores the file temporarily through Multer.
  3. The analyze endpoint reads headers and sample rows.
  4. The convert endpoint processes rows and writes a CSV file.
  5. The frontend receives preview data and a download URL.
  6. Temporary files are removed after processing and old files are cleaned on an interval.
